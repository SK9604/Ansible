1. RAID 개요와 특징

- RAID(레이드)란 Redundant Array of Inexpensive(또는 Independent) Disks의 약자로 저렴한 (또는 독립된) 디스크의 복수 배열을 뜻한다.
- 최초 목적은 저렴한 저용량의 디스크 여러 개를 하나의 대용량의 디스크로 만들어 사용하는 것이었다.
- 파티션은 하나의 하드디스크를 논리적으로 분할하는 기술이고, RAID는 여러 하드디스크를 논리적으로 하나의 디스크로 만드는 기술이다.
- 1TB 하드디스크 10개를 가지고 마치 10TB 하드디스크 1개처럼 사용할 수 있다.
- 하드웨어 RAID와 소프트웨어 RAID 두 가지로 분류한다.
- 소프트웨어 RAID는 컴퓨터 내에 장착된 여러 개의 디스크를 RAID로 구성한다.
- 윈도우 서버는 RAID 기능을 기본적으로 지원한다.
- 소프트웨어 RAID는 하드웨어 RAID보다 안정성이 떨어지기는 하지만 디스크만 추가하면 빠르게 안정적인 시스템을 구성할 수 있다.

2. RAID 레벨(Level) 구성도

- 단순 볼륨: 디스크 1개로 1개의 볼륨
- 스팬 볼륨(Spanned Volume): 2개 이상으로 구성. 첫 디스크에 데이터가 모두 찬 후에 두 번째 디스크를 사용. 디스크의 크기가 달라도 공간 효율이 100%
- 스트라이프 볼륨(Striped Volume: RAID 0): 2개 이상으로 구성. 동시 저장하는 방식이므로 입출력 속도가 꽤 향상됨. 안정성은 떨어짐. 디스크의 크기가 같아야 공간 효율 100%
- 미러 볼륨(Mirrored Volume: RAID 1): 거울처럼 똑같은 두 개의 디스크를 구성. 안정성이 뛰어나지만, 비용이 두배로 소요됨
- RAID-5: 안전성이 어느 정도 보장되면서 공간 효율성도 좋은 방식. 패리티 비트 방식을 사용하며 최소 3개 이상의 디스크 필요함. RAID-4는 하나의 디스크에 패리티 비트를 저장하는 반면 RAID-5는 모든 디스크에 패리티 비트를 저장하는 방식
- RAID Level 2~4는 현재 거의 사용하지 않는다.
- RAID Level 6는 최소 4개의 디스크가 필요하며, 2개의 디스크에 패리티를 구성하여 최대 2개의 디스크 장애 시에도 데이터 안정성을 보장한다. 성능보다는 안정성에 중점

3. RAID 구성

- 하드디스크 추가 장착 구성도
- FIRST에 하드디스크를 추가 장착한다. (TB가 아닌 GB로 실습)
- 디스크 관리
- 디스크 온라인 및 초기화
- 동적 디스크(RAID를 구성하려면 동적 디스크로 변환해야 한다.)
- 스팬 볼륨 구현
- 스트라이프 볼륨 구현
- 미러 볼륨 구현
- RAID-5 구현

4. 디스크 고장 시 RAID

- 시스템을 종료한 후 하드웨어 구성에서 디스크 일부(X표시)를 제외 시킨다.
- 다시 부팅 후 윈도우 탐색기
- 스팬 볼륨과 스트라이프 볼륨은 윈도우 탐색기에 보이지 않지만, 미러 볼륨과 RAID-5 볼륨은 남아 있다.
- 부팅 후 디스크 관리자
- 중복 실패는 데이터는 보족되지만 결함 허용을 지원하지 못하는 상태를 말한다.
- 현재 데이터가 살아 있지만 중복되지 않아 안전한 상태가 아니다.

5. RAID 디스크 복구

- 미러 볼륨과 RAID-5는 복구 후 결함을 허용한다.
- 스팬 볼륨과 스트라이프 볼륨은 상태를 복원할 수 없다. 고장난 디스크르 데이터 복구 서비스를 받아야 한다.
- 다시 새 하드디스크 장착
- 디스크 관리 (자동으로 디스크 초기화를 시작)
- 동적 디스크
- 스팬 볼륨 및 스트라이프 볼륨 삭제
