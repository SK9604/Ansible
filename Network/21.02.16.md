# NAT(Network Address Translation)

### NAT의 개요

1. 원래의 IP주소를 다른 것으로 변환하여 전송하는 것을 말한다.
2. 주로 사설 IP주소(Private IP Address)를 공인 IP주소(Public IP Address)로 변환시켜 인터넷으로 전송하고, 인터넷에서 수신한 IP 패킷을 내부망으로 전송할 때 사설 IP주소(Private IP Address)로 변환하기 위해 사용된다.
3. 사설 IP Address 범위
    1. Class A : 10.0.0.0 ~ 10.255.255.255
    2. Class B : 172.16.0.0 ~ 172.31.255.255
    3. Class C : 192.168.0.0 ~ 192.168.255.255

### NAT의 특징과 목적

1. 공인 IP주소(Public IP Address)의 절약을 위해 대부분의 기관에서 NAT를 사용하고 있다.
2. NAT를 사용하면 내부망에서 사용하는 사설 IP주소(Private IP Address)를 일반적으로 외부에서는 알 수 없어 네트워크 보안이 강화된다.
3. 네트워크 관리자는 IP주소 할당을 사설 IP주소(Private IP Address) 범위 안에서 할당함으로 효율적인 주소 할당이 가능해 진다.
4. 라우터에서 효율적인 축약이 가능하여 성능이 향상되고 장애처리가 쉬워진다.
5. 공인 IP주소(Public IP Address)가 부여되어야만 하는 특정 Service가 있다.
