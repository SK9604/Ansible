# 용어 정리

### 네트워크

Net(그물)+Work(일) = (컴퓨터를 이용한) 공동작업

네트워크의 일반적인 목적은 자료와 주변장치를 서로 공유하는 것이며, 능률적인 업무와 데이터 백업관리를 지원한다.

### 인터넷

서로 다른 네트워크라는 의미이고, 전 세계의 컴퓨터가 서로 연결되어 정보를 교환할 수 있는 거대한 컴퓨터 통신망이다.

인터넷은 TCP/IP라는 표준 프로토콜로 서로 연결되어 있다.

### 데이터

인간이나 컴퓨터가 처리할 수 있도록 실제 생활의 많은 일들을 간단히 문자, 숫자, 기호 등으로 표현

### 정보

특정 목적을 위해 의미를 부여한 데이터 (데이터를 가공하여 의미부여)

### 정보통신

정보전송과 정보처리 기능이 통합된 형태를 의미하고 정보의 공유, 변환, 전송이 가능하다

■ 3대 목표

- 정확성 : 정보가 전송 도중 감쇠, 잡음 등으로 인해 손실되거나 변형되지 않아야 한다.
- 효율성 : 투자된 장비나 비용에 비해 정보의 가치가 커야 한다.
- 보안성 : 정보가 제 3자에게 유출되지 않도록 기밀을 유지해야 한다.

■ 3 요소

- 정보원(송신자) : 정보를 만들어서 보내는 사람이나 기계 (Source)
- 전송매체(전송기기) : 정보를 전송하는 매체, 통신회선
- 정보처리원(수신자) : 정보를 받아서 처리하는 사람이나 기계 (Destination)

### 프로토콜

네트워크 장비들을 통해 각 컴퓨터들이 데이터를 주고 받을 수 있도록 하는 규칙(약속)

■ 3 요소

- 구문(Syntax) : 데이터의 구조나 형식, 부호화, 신호의 크기
- 의미(Semantics) : 조정과 오류 관리를 위한 제어 정보
- 타이밍(Timing) : 속도와 순서

### 정보와 전송 방식

- 디지털 전송 : 디지털 신호를 사용해서 데이터를 전송. 아날로그 전송에 비해 잡음 등에 강하다.

- 브로드밴드(Broadband)

## IP Addressing

### IP Address

일반적으로 IPv4 Address를 말하며 인터넷에서 사용되는 각종 장비들(라우터, 스위치, 서버, PC등)을 구분하기 위해 부여되는 논리적인 주소이다.

IANA → APNIC → KISA → ISP → User 순으로 할당된다.

IPv4 Address는 1983년~2011년 2월까지 할당되었으며 지금은 IPv4 주소의 고갈 문제를 해결하기 위해 IPv6 Address가 표준화되어 대체되고 있다.

### IP Address의 표기와 특징

IPv4 Address는 32비트로 되어 있으며 8비트 단위로 점을 찍어 구분한다.

비트로 표기하면 사용하기 불편하므로 0~255까지의 정수로 표기한다.

IPv4 Address는 네트워크 부분과 호스트 부분으로 나뉜다.

같은 네트워크 안에 속한 장비 간의 통신이 되려면 네트워크 부분은 반드시 서로 일치해야 하고, 호스트 부분은 반드시 서로 달라야 한다. IP 주소의 네트워크 부분이 다른 장비들 사이의 통신은 라우터가 반드시 필요하고, 네트워크 부분이 같은 장비들 사이의 통신은 라우터를 사용할 수 없다.

네트워크 부분과 호스트 부분을 구분하기 위해 서브넷 마스크를 함께 사용한다.

IPv4 Address는 Class A ~ Class E까지 5개의 클래스로 분류하며 각 클래스는 용도가 다르다.

주 네트워크(Major Network)에 기반한 Classful과 주 네트워크에 기반하지 않는 Classless의 개념을 갖는다.

공인 주소와 사설 주소가 있다.

하나의 네트워크 안에는 Network Address, Broadcast Address, Assigned Address가 있다.

Network Address: 해당 네트워크의 가장 첫 번째 IP Address로서 해당 네트워크를 대표할 때 사용되는 IP Address이다. 네트워크의 형태에 따라 Network Name, Network Number, Network ID, Subnet Address, Subnet Name, Subnet Number 등 다양하게 불려진다. 호스트 부분의 모든 비트가 0인 IP Address이다.

Broadcast Address: 해당 네트워크의 맨 마지막 IP Address로서 해당 네트워크 내의 다른 호스트들에게 무언가를 전송하고자 할 때 목적지 IP Address로 사용한다. 호스트 부분의 모든 비트가 1인 IP Address이다.

Assigned Address: 실제 호스트에 할당 가능한 IP Address이다.

Network Address와 Broadcast Address는 호스트의 인터페이스에 할당할 수 없다.

### IP Address Classes

Class A : 1 Octet의 첫 번째 비트가 0 인 주소. Class A는 앞 8비트가 네트워크 부분이고 뒤 24비트가 호스트 부분이다.

Class B : 1 Octet의 처음 두 비트가 10 인 주소. Class B는 앞 16비트가 네트워크 부분이고 뒤 16비트가 호스트 부분이다.

Class C : 1 Octet의 처음 세 비트가 110 인 주소. Class C는 앞 24비트가 네트워크 부분이고 뒤 8비트가 호스트 부분이다.

Class D : 1 Octet의 처음 네 비트가 1110 인 주소. Class D는 네트워크와 호스트의 구분이 없으며 멀티캐스트(Multicast)용으로 사용한다. 

Class E : 1 Octet의 처음 네 비트가 1111인 주소. Class E는 네트워크와 호스트의 구분이 없으며 연구용 또는 실험용으로 사용한다.

### 사설 IP 주소(Private IP Address)

사설 IP 주소는 인터넷 용도로 사용할 수 없으며 조직 내부에서만 사용되는 IP 주소이다.

Class A,B,C의 각각 한 부분이 이에 해당한다.

### 공인(Public) 제외 주소들

0.0.0.0 ~ 0.255.255.255: 첫 번째 Octet이 0인 IP 주소는 전체 IP 주소를 의미한다.

127.0.0.0 ~ 127.255.255.255: 장비 자신을 뜻하는 루프백(Loopback) 주소이다.

255.255.255.255: Local Broadcast Address

Private Address: 사설 주소는 Internet Interface에 할당하면 안 된다.

### Subnetting의 개요

IP Address의 고갈 문제를 해결하기 위해 1985년에 <RFC 950.인터넷 표준 서브넷팅 절차>를 통해 Subnetting의 개념을 설계하여 발표하였다.

Subnetting은 하나의 주 네트워크(Major Network)를 필요한 만큼 분할하여 상호 연결 망을 구축할 수 있게 해 준다.

Subnetting은 Classful two-level addressing hierarchy로써 Routing Table의 증가 문제의 효율적인 해결방안이 되었다.

라우터는 네트워크에 Packet을 전송할 때 목적지 IP 주소의 네트워크 부분만 심사하여 해당 Interface로 전송한다.

라우터의 Interface가 다르면 부여하는 IP 주소의 네트워크 부분도 반드시 달라야 한다.

### Subnetting의 장점

Broadcast Size를 줄여 주므로 LAN의 Traffic을 줄일 수 있다.

쓸모 없이 버려지는 IP 주소를 최소화 할 수 있다.

여러 네트워크로 분할되므로 각 Subnetwork 간에 보안을 유지할 수 있다.

### Subnet Mask

Subnet Mask는 IP Address의 첫 비트부터 어디까지가 네트워크 부분인지를 알려주는 역할을 한다.

IP Address와 마찬가지로 32비트로 되어 있으며 네트워크 부분의 비트는 1, 호스트 부분의 비트는 0이다.

IP Address 다음에 표시하며 점으로 구분된 십진수를 사용한다.

네트워크 비트는 연속성이 있으며 호스트 부분 중간에 들어갈 수 없다.

Class A는 8비트, Class B는 16비트, Class C는 24비트의 Default Subnet Mask를 갖는다.

### Subnetting 방법

호스트 부분의 비트 일부를 네트워크 비트화 시킨다.

IP Address 192.168.1.1은 Class C이며 Default Subnet Mask는 24비트다. 11000000.10101000.00000001.00000001 (IP Address) 11111111.11111111.11111111.00000000 (Subnet Mask) -------------------------------------------- (And 연산) 11000000.10101000.00000001.00000000 (Network Address) 11000000.10101000.00000001.11111111 (Broadcast Address) 즉, 192.168.1.1은 192.168.1.0~192.168.1.255에 속한 IP 주소다.

Subnet Mask의 길이를 기본 24비트에서 25비트 혹은 26비트로 바꾸면 변화된 비트 수 만큼 네트워크가 분할된다. 아래의 표는 4번째 Octet의 Subnet Mask 길이 변화 시 Subnet 계산 방법이다.

```bash
┌============================================================================================┐
| subnet Mask | Subnet 길이(n) | Subnet 수(2^n) | Host 길이(x) | Host 수(2^x-2) | Subnet 주소 |
├=============+================+================+=============+================+=============┤
|  1000 0000  |        1       |       2        |      7      |      126       |   0, 128    |
├-------------+----------------+----------------+-------------+----------------+-------------┤
|  1100 0000  |        2       |       4        |      6      |      62        |0,64,128,192 |
├-------------+----------------+----------------+-------------+----------------+-------------┤
|  1110 0000  |        3       |       8        |      5      |      30        |0,32,64,96,..|
├-------------+----------------+----------------+-------------+----------------+-------------┤
|  1111 0000  |        4       |       16       |      4      |      14        |0,16,32,48,..|
├-------------+----------------+----------------+-------------+----------------+-------------┤
|  1111 1000  |        5       |       32       |      3      |      6         |0,8,16,24,...|
├-------------+----------------+----------------+-------------+----------------+-------------┤
|  1111 1100  |        6       |       64       |      2      |      2         |0,4,8,12,16,.|
├-------------+----------------+----------------+-------------+----------------+-------------┤
|  1111 1110  |        7       |       128      |      1      |      0         |0,2,4,6,8,...|
├-------------+----------------+----------------+-------------+----------------+-------------┤
|  1111 1111  |        8       |       256      |      0      |                |0,1,2,3,4,...|
└============================================================================================┘
```

결과적으로 Subnetting을 하면 Subnetwork의 수는 증가하고, Subnetwork 당 호스트의 수는 감소한다.
