# QUESTION NO: 381

### 솔루션 아키텍트는 Windows 사용자의 홈 디렉토리를 위한 탄력적인 솔루션을 설계해야 합니다. 솔루션은 회사의 Active Directory를 기반으로 내결함성, 파일 수준 백업 및 복구, 액세스 제어를 제공해야 합니다.
이러한 요구 사항을 충족하는 스토리지 솔루션은 무엇입니까?

**A. 사용자의 홈 디렉터리를 저장하도록 Amazon S3를 구성합니다. Amazon S3를 Active Directory에 가입합니다.
B. Windows 파일 서버용 Amazon FSx를 사용하여 다중 AZ 파일 시스템 구성 Amazon FSx를 Active Directory에 가입시킵니다.
C. 사용자의 홈 디렉터리에 대해 Amazon Elastic File System (Amazon EFS)을 구성합니다. Active Directory로 AWS Single Sign-On을 구성합니다.
D. 사용자의 홈 디렉터리를 저장하도록 Amazon Elastic Block Store (Amazon EBS) 구성 Active Directory로 AWS Single Sign-On을 구성합니다.**

- 키워드

    Windows, 내결함성, 파일 수준 백업 및 복구, 액세스 제어

- 답 : `B`
- 오답 정리

    A - windows 홈 디렉토리와 관련 X

    C - EFS는 윈도우를 지원하지 않는다. 리눅스 지원

    D - windows 홈 디렉토리와 관련 X

- 서비스 정리

    **[Amazon FSx for Windows File Server](https://aws.amazon.com/ko/fsx/windows/faqs/?nc1=h_ls)** : Windows 파일 서버용 Amazon FSX는 업계 표준 SMB(Service Message Block) 프로토콜을 통해 액세스할 수 있는 완벽하게 관리되고 안정적이며 확장 가능한 파일 스토리지를 제공합니다. Windows Server를 기반으로 구축되어 사용자 할당량, 최종 사용자 파일 복원, 마이크로소프트 AD(Active Directory) 통합과 같은 광범위한 관리 기능을 제공합니다.

    **Active Directory** : 회사 직원들의 계정 정보와 컴퓨터에 대한 정보, 그리고 회사에서 강제하고자 하는 정책들에 대한 정보를 저장하고 있는 일종의 데이터베이스

# QUESTION NO: 382

### 회사가 서비스를 사용하여 2 계층 전자 상거래 웹 사이트를 운영하고 있습니다. 현재 아키텍트는 프라이빗 서브넷의 Amazon EC2 인스턴스로 트래픽을 전송하는 게시 관련 Elastic Load Balancer를 사용합니다. 정적 컨텐츠는 EC2 인스턴스에서 호스팅되며 동적 컨텐츠는 MYSQL 데이터베이스에서 검색됩니다. 응용 프로그램이 미국에서 실행 중입니다. 이 회사는 최근 유럽과 호주 사용자에게 판매를 시작했습니다. 솔루션 아키텍트는 솔루션을 디자인하여 해외 사용자가 향상된 브라우징 경험을 갖도록 해야 합니다.
가장 비용 효율적인 솔루션은 무엇입니까?

**A. Amazon S3에서 전체 웹 사이트를 호스팅합니다.
B. Amazon CloudFront 및 Amazon S3를 사용하여 정적 이미지를 호스팅하십시오.
C. 퍼블릭로드 밸런서 및 EC2 인스턴스 수 증가
D. 유럽 및 Austraila의 AWS 리전에서 2 계층 웹 사이트를 배포합니다.**

- 키워드

    정적 컨텐츠, 동적 컨텐츠 모두 사용. 해외 사용자가 향상된 브라우징 경험을 갖도록

- 답 : `B`
- 오답 정리

    A - CloudFront의 데이터 전송 비용은 S3보다 낮습니다. 정적 컨텐츠의 읽기 작업이 많은 경우 S3 버킷 앞에 CloudFront를 추가하는 것이 더욱 경제적입니다.

    C - 퍼블릭 로드 밸런서가 무엇인가..?

    D - 하나의 방법이 될 수 있으나 비용 효율적인 측면에서 탈락

**💡 CloudFront = 정적 컨텐츠만?**

NO. 새 코드를 작성하지 않고 추가 비용 없이 개인화된 동적 웹 콘텐츠 제공

Amazon ELB(Elastic Load Balancer) 또는 Amazon EC2 인스턴스에서 인터넷상의 최종 사용자에게 직접 웹 애플리케이션 또는 API와 같은 동적 콘텐츠를 제공하는 경우 Amazon CloudFront를 콘텐츠 전송 네트워크로 사용하여 콘텐츠의 성능, 가용성 및 보안을 향상시킬 수 있습니다. Amazon CloudFront를 사용하면 최종 사용자 연결이 그들에게 더 가까운 CloudFront 위치에서 종료되므로 연결 설정에 필요한 전체 왕복 시간을 줄이는 데 도움이 됩니다. 이러한 CloudFront 위치는 AWS 오리진에 연결하기 위한 뛰어난 성능 및 가용성을 제공하는 복원성이 높은 Amazon Backbone Network에 연결되어 있습니다. 또한, 오리진에 대한 지속적인 TCP 연결, 세션 티켓 및 OCSP 스테이플링과 같은 SSL 보강 등의 기타 다양한 최적화는 캐시가 불가능한 동적 콘텐츠의 성능 개선에도 도움이 됩니다.
