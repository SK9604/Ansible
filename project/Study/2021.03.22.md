# QUESTION NO: 365

### 회사는 사용자 데이터를 AWS에 저장합니다. 데이터는 업무 시간 동안 최대 사용량으로 지속적으로 사용됩니다. 액세스 패턴은 다양하며 일부 데이터는 한 번에 몇 달 동안 사용되지 않습니다. 솔루션 설계자는 고 가용성을 유지하면서 최고 수준의 내구성을 유지하는 비용을 선택해야 합니다.
이러한 요구 사항을 충족하는 스토리지 솔루션은 무엇입니까?

**A. Amazon S3 표준
B. Amazon S3 지능형 계층화
C. Amazon S3 Glacier 딥 아카이브
D. Amazon S3 One Zone-Infrequent Access (S3 One Zone-IA)**

- 키워드

    지속적인 사용, 다양한 액세스 패턴, 일부 데이터는 몇 달 동안 사용되지 않음, 고가용성, 내구성

- 답 : `B`
- 오답 정리

    A - 질문의 내용에 사용할 수는 있으나 액세스 패턴이 다양하다는 점에서 B가 조금 더 적합하다고 볼 수 있다.

    C - 지속적으로 사용되는 데이터에 적합하지 않음

    D - 고 가용성에 적합하지 않음 (one zone)

# QUESTION NO: 366

### 회사의 IT 비용에 대한 최근 분석에 따르면 백업 비용을 줄여야 합니다. 이 회사의 최고 정보 책임자는 물리적(실제) 백업 테이프 사용을 제거하여 온-프레미스 백업 인프라를 단순화하고 비용을 절감하고자 합니다. 회사는 온-프레미스 백업 응용 프로그램 및 워크 플로에 대한 기존 투자를 유지해야 합니다.
솔루션 아키텍트는 무엇을 추천해야 합니까?

**A. NFS 인터페이스를 사용하여 백업 애플리케이션에 연결하도록 AWS Storage Gateway를 설정하십시오.
B. NFS 인터페이스를 사용하여 백업 애플리케이션과 연결되는 Amazon EFS 파일 시스템 설정
C. iSCSI 인터페이스를 사용하여 백업 애플리케이션과 연결되는 Amazon EFS 파일 시스템 설정
D. iSCSI 가상 테이프 라이브러리 (VTL) 인터페이스를 사용하여 백업 애플리케이션에 연결하도록 AWS Storage Gateway를 설정하십시오.**

- 키워드

    백업 테이프(Backup tapes)

- 답 : `D`
- 오답 정리

    물리적 백업 테이프를 제거하고 기존 워크 플로를 유지하는 작업은 VTL 뿐이다.

- 서비스 정리

    **[VTL](https://aws.amazon.com/ko/storagegateway/vtl/)(Virtual Tape Library)** : 테이프 게이트웨이를 사용하면 기존 백업 워크플로를 변경하지 않아도 온프레미스에서 물리적 테이프를 AWS에서 가상 테이프로 대체할 수 있습니다. 테이프 게이트웨이는 모든 주요 백업 애플리케이션을 지원하며, 데이터 액세스 시 지연 시간을 낮추도록 온프레미스에서 가상 테이프를 캐시합니다. 테이프 게이트웨이는 게이트웨이와 AWS 간의 데이터를 암호화하여 데이터 전송을 보호하고, 데이터를 압축하고 Amazon S3와 Amazon S3 Glacier 또는 Amazon S3 Glacier Deep Archive 간에 가상 테이프를 전환하여 스토리지 비용을 최소화합니다.

    ![https://s3-us-west-2.amazonaws.com/secure.notion-static.com/5610d807-d2b0-49a8-8f94-26e5e1cda9fc/Untitled.png](https://s3-us-west-2.amazonaws.com/secure.notion-static.com/5610d807-d2b0-49a8-8f94-26e5e1cda9fc/Untitled.png)

    **[테이프 게이트웨이](https://docs.aws.amazon.com/ko_kr/storagegateway/latest/userguide/StorageGatewayConcepts.html#storage-gateway-vtl-concepts)** : 데이터를 내구성이 있고 비용 효율적인 방식으로 AWS 클라우드에 보관할 수 있는 솔루션을 제공합니다. 가상 테이프 라이브러리(VTL) 인터페이스를 통해 기존 테이프 기반 백업 인프라를 최대한 활용하여 테이프 게이트웨이에 생성하는 가상 테이프 카트리지에 데이터를 저장할 수 있습니다. 각 테이프 게이트웨이는 미디어 체인저 및 테이프 드라이브로 미리 구성됩니다. 이러한 테이프 드라이브와 미디어 체인저는 기존 클라이언트 백업 애플리케이션에서 iSCSI 디바이스로 사용할 수 있습니다. 필요에 따라 테이프 카트리지를 추가하여 데이터를 보관합니다.

    **[AWS Storage Gateway](https://docs.aws.amazon.com/ko_kr/storagegateway/latest/userguide/WhatIsStorageGateway.html)** : 온프레미스 소프트웨어 어플라이언스를 클라우드 기반 스토리지와 연결하여 온프레미스 IT 환경과 AWS 스토리지 인프라스트럭처. 서비스를 사용하여 데이터를 AWS 데이터 보안을 유지하는 데 도움이 되는 확장 가능하고 비용 효율적인 스토리지를 위한 클라우드.

    **[NFS](https://sectionr0.tistory.com/45)** : 클라이언트 호스트가 원격 서버의 파일에 마치 로컬 파일처럼 접근할 수 있게 하기 위해 개발 됨. 성능, 단순성, 제조업체 간 호환성을 주요 목표로 설정됨. 네트워크 상의 파일을 직접 연결된 스토리지에 접근하는 방식과 비슷한 방식으로 접근하도록 도와줌

    **[iSCSI](https://otsteam.tistory.com/256)** : 데이터 기억 장치를 연결하기 위한 새로운 IP 기반 기억 장치 네트워킹 표준.

    **[EFS](https://aws.amazon.com/ko/efs/)** : 스토리지를 프로비저닝하거나 관리하지 않고도 파일 데이터를 공유할 수 있게 해주는 간단하고 한 번만 설정하면 되는 탄력적 서버리스 파일 시스템을 제공합니다.

**💡 백업 테이프?**

테이프 백업은 하드 Disk가 손상되거나 오류가 발생한 경우 데이터를 복구할 수 있도록 주 저장 장치에서 테이프 카트리지로 정기적으로 복사하는 작업입니다.
