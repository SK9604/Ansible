## KOPS vs EKS

1. 클라우드 지원

    EKS : AWS 사용자와 기업만을 위함

    KOPS : AWS는 공식 지원, GCE와 OpenStack은 베타 지원, VMware vSphere는 알파 지원

2. 보안

    EKS : 마스터 노드를 EKS가 관리하기 때문에 사용자는 워커 노드에 대한 보안만 관리

    KOPS : 사용자가 모두 관리해야 함

3. 비용

    EKS : 클러스터당 시간당 $0.2

    KOPS : 대규모 클러스터의 경우 리소스가 많은 마스터 노드가 필요, 자체 VPC를 생성하므로 VPC 비용 추가 -> 대규모일수록 비용이 매우 증가함

4. 운영 노력

    EKS : 제어 영역을 자동 복구하기 때문에 마스터 노드에 대한 모니터링/경고가 필요하지 않다.

    KOPS : 마스터 노드에 문제가 있는 경우 사용자는 경고를 모니터링하고 문제를 해결해야 함

5. 클러스터 조정

    EKS : 마스터 노드에 양호한 기본값을 제공하지만 미세 조정할 수 없다.

    KOPS : EKS에 비해 미세한 조정을 할 수 있다.
